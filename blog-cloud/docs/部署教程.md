# Blog Cloud 部署教程

Spring Cloud 微服务博客的 CentOS 7 部署步骤。

## 一、环境要求

- CentOS 7
- JDK 8
- Node.js 16+、npm
- 已编译好的 jar 包（4 个，不纳入 Git 仓库）

## 二、端口规划

| 服务 | 端口 | 说明 |
|------|------|------|
| Eureka | 8761 | 服务注册中心 |
| blog-gateway | 9000 | API 网关 |
| blog-home | 9001 | 博客首页 API |
| blog-admin | 9002 | 后台管理 API |
| Vue 前端 | 8082 | 用户访问入口 |

## 三、部署目录

建议在 `/opt/web` 下按如下结构部署：

```
/opt/web/
├── eureka/
│   ├── eureka-0.0.1-SNAPSHOT.jar
│   └── application.yml
├── blog-home/
│   ├── blog-home-1.0.0.jar
│   └── application.yml
├── blog-admin/
│   ├── blog-admin-1.0.0.jar
│   └── application.yml
├── blog-gateway/
│   ├── blog-gateway-1.0.0.jar
│   └── application.yml
├── blog-ui/blog-ui/          # Vue 前端
└── start-all.sh              # 一键启动脚本
```

## 四、配置要点

### 1. Eureka 注册 IP

各微服务的 `application.yml` 中建议配置：

```yaml
eureka:
  instance:
    prefer-ip-address: true
    ip-address: 127.0.0.1
```

避免使用主机名 `Centos7` 注册，否则 Gateway 的 Netty DNS 可能解析失败导致 500。

### 2. 网关 CORS

修改 `blog-gateway/application.yml` 中的 `allowedOrigins` 为你的实际 IP：

```yaml
allowedOrigins: "http://192.168.x.x:8082"
```

### 3. 前端代理

`frontend/vue.config.js` 中前端端口和代理目标：

```javascript
devServer: {
  port: 8082,
  proxy: {
    '/': {
      target: 'http://localhost:9000',
      ...
    }
  }
}
```

## 五、启动顺序

```
Eureka → blog-home / blog-admin → blog-gateway → 前端
```

手动启动：

```bash
cd /opt/web/eureka && nohup java -jar eureka-0.0.1-SNAPSHOT.jar > nohup.out 2>&1 &
sleep 10
cd /opt/web/blog-home && nohup java -jar blog-home-1.0.0.jar > nohup.out 2>&1 &
cd /opt/web/blog-admin && nohup java -jar blog-admin-1.0.0.jar > nohup.out 2>&1 &
sleep 10
cd /opt/web/blog-gateway && nohup java -jar blog-gateway-1.0.0.jar > nohup.out 2>&1 &
sleep 10
cd /opt/web/blog-ui/blog-ui && npm run serve
```

或使用一键脚本：

```bash
/opt/web/start-all.sh
```

## 六、验证

```bash
# 检查端口
ss -tlnp | grep -E ':(8761|9000|9001|9002|8082)'

# 检查 API
curl http://localhost:9000/home/api/posts
curl http://localhost:9000/admin/posts
```

浏览器访问 `http://IP:8082/` 应能看到文章列表。

## 七、常见问题

| 现象 | 原因 | 解决 |
|------|------|------|
| 前端空白 | 网关 500 | 检查 Eureka 注册 IP 是否为 127.0.0.1 |
| 9000 根路径 404 | 网关无首页路由 | 正常，请访问 8082 |
| 端口被占用 | 上次进程未退出 | `ss -tlnp \| grep 端口` 后 kill |
