# resume项目部署

> 本仓库已整理为展示用结构：`docs/` 部署教程、`sql/` 数据库脚本、`webapp/` 项目文件。  
> 部署到 Tomcat 时，将 `webapp/` 目录内容复制到 `/usr/local/tomcat8/webapps/` 下即可。

### 一、环境准备

准备一台服务器，2核心2线程，2G的服务器；

我们一般有了新的项目要部署就会申请服务器，一般我们拿到一台新的服务器都要对这个服务器进行一个初始化，比如IP的配置，防火墙的配置，还有账号密码的设置等，我这里搞了一个简单的初始化脚本

```bash
#!/bin/bash
# 设置要分配的 IP 地址、子网掩码和默认网关（按你的 VMware 网段修改）
ip_address="192.168.153.8"
subnet_mask="255.255.255.0"
default_gateway="192.168.153.2"

echo '正在关闭防火墙'
systemctl stop firewalld && systemctl disable firewalld

echo '正在关闭 NetworkManager'
systemctl stop NetworkManager && systemctl disable NetworkManager

echo '正在关闭 selinux'
setenforce 0 && sed -ri 's/SELINUX=enforcing/SELINUX=disabled/g' /etc/selinux/config

echo '正在配置网卡'
# 自动检测网卡名（VMware 常见 ens33/ens32，旧版可能是 eth0）
nic=$(ls /sys/class/net/ | grep -vE '^(lo|docker|virbr|veth)' | head -1)
[ -z "$nic" ] && nic="ens32"
networkfile="/etc/sysconfig/network-scripts/ifcfg-${nic}"

cat > "$networkfile" <<EOF
TYPE="Ethernet"
PROXY_METHOD="none"
BROWSER_ONLY="no"
BOOTPROTO="static"
DEFROUTE="yes"
IPV4_FAILURE_FATAL="no"
IPV6INIT="no"
NAME="${nic}"
DEVICE="${nic}"
ONBOOT="yes"
IPADDR=${ip_address}
NETMASK=${subnet_mask}
GATEWAY=${default_gateway}
DNS1=223.5.5.5
DNS2=114.114.114.114
EOF

echo "网卡配置文件: ${networkfile}，设备名: ${nic}"

echo '正在重启网络（yum 安装前必须先通网）'
systemctl enable network
systemctl restart network
sleep 3

# 写入 DNS，避免 resolv.conf 为空导致无法解析域名
cat > /etc/resolv.conf <<EOF
nameserver 223.5.5.5
nameserver 114.114.114.114
EOF

echo '正在修复 CentOS 7 yum 源（官方 mirrorlist 已下线，需改用 vault 镜像）'
sed -i 's/mirrorlist/#mirrorlist/g' /etc/yum.repos.d/CentOS-*.repo
sed -i 's|#baseurl=http://mirror.centos.org/centos/\$releasever|baseurl=https://mirrors.aliyun.com/centos-vault/7.9.2009|g' /etc/yum.repos.d/CentOS-*.repo
yum clean all
yum makecache fast

echo '正在安装基础软件包'
yum -y install wget lrzsz unzip gzip telnet vim

echo '配置基础环境变量'
cat >> /etc/profile <<'EOF'
if [ $(id -u) == 0 ]; then
export PS1='[\[\e[31;1m\]\u\[\e[m\]@\h \w]# '
else
export PS1='[\[\e[34;1m\]\u\[\e[m\]@\h \w]$ '
fi
alias ll='ls -lh --color=auto'
alias grep='grep --color'
alias free='free -m'
alias df='df -h'
alias du='du -h'
alias vi='vim'
EOF
cat >> ~/.vimrc <<'EOF'
"设置高亮
syntax on
"显示行号
set number
"自动缩进功能
set autoindent
"启用鼠标支持
"set mouse=a
"vim新打开.sh文件自动添加一下内容
autocmd BufNewFile *.sh exec ":call SetComment()"
func SetComment()
     if expand("%:e")=='sh'
         call setline(1,'#!/bin/bash')
         call setline(2,'#Author:     xxx')
         call setline(3,'#Create time:'.strftime("%Y-%m-%d"))
     endif
endfunc
EOF
source /etc/profile
echo '脚本执行完毕'
echo "当前 IP: $(ip -4 addr show ${nic} | grep -oP '(?<=inet\s)\d+(\.\d+){3}')"
ping -c 2 ${default_gateway} && echo '网关连通正常' || echo '警告: 网关不通，请检查 VMware 网段与 ip_address/gateway 是否匹配'
```

没有执行权限，给他一个执行权限

```
chmod +x init.sh
```

> **常见报错说明**
>
> 1. `Could not resolve host: mirrorlist.centos.org`：CentOS 7 已停止维护，官方 mirrorlist 失效，需改用 vault 镜像（脚本已内置修复）。
> 2. `Cannot find a valid baseurl for repo: base/7/x86_64`：同上，换源后执行 `yum clean all && yum makecache`。
> 3. `network.service failed`：原脚本在 **网络未生效时就执行 yum**，且网卡名可能不是 `ens32`。新脚本会自动检测网卡并在 yum 前重启 network。
> 4. 若 VMware 网段不是 `192.168.153.x`，请修改脚本开头的 `ip_address` 和 `default_gateway`（在 VMware「虚拟网络编辑器」中查看 VMnet 网段）。
>
> **已在报错虚拟机上手动修复（无需重装）：**
> ```bash
> # 1. 查看实际网卡名
> ip a
> # 2. 换 yum 源
> sed -i 's/mirrorlist/#mirrorlist/g' /etc/yum.repos.d/CentOS-*.repo
> sed -i 's|#baseurl=http://mirror.centos.org/centos/\$releasever|baseurl=https://mirrors.aliyun.com/centos-vault/7.9.2009|g' /etc/yum.repos.d/CentOS-*.repo
> yum clean all && yum makecache fast
> # 3. 确认 DNS 和网络
> echo -e "nameserver 223.5.5.5\nnameserver 114.114.114.114" > /etc/resolv.conf
> systemctl restart network
> ping -c 2 223.5.5.5
> # 4. 再安装软件
> yum -y install wget lrzsz unzip gzip telnet vim
> ```

##### 3.配置

安装完成后我们不要急着启动，先给mysql来一个自定义配置

备份原来的配置，然后添加新的配置

```bash
mv /etc/my.cnf /etc/my.cnf.bak.$(date +%Y%m%d)

# 推荐用 cat 直接写入，避免 vim 粘贴带进 ``` 等非法字符
cat > /etc/my.cnf <<'EOF'
# 客户端连接配置
[client]
# 本地连接使用的 Unix 套接字文件
socket=/var/lib/mysql/mysql.sock

# MySQL 服务端配置
[mysqld]
# 监听端口，默认 3306
port=3306
# 数据文件存储目录
datadir=/var/lib/mysql
# 服务端套接字文件路径
socket=/var/lib/mysql/mysql.sock
# 服务器唯一标识，主从复制时需要
server-id=1
# 开启二进制日志，主从复制时需要
log-bin=mysql-bin
# 禁用符号链接，防止安全风险
symbolic-links=0
# 错误日志路径
log-error=/var/log/mysqld.log
# 服务进程 PID 文件路径
pid-file=/var/run/mysqld/mysqld.pid
# 服务端默认字符集
character_set_server=utf8mb4
# 服务端默认排序规则
collation_server=utf8mb4_general_ci

# 加载 /etc/my.cnf.d/ 目录下的额外配置
!includedir /etc/my.cnf.d/
EOF

# 启动前校验配置文件（无输出表示格式正确）
my_print_defaults --defaults-file=/etc/my.cnf --mysqld >/dev/null && echo "my.cnf 格式正确"
```

如需手动编辑，可用 `vim /etc/my.cnf`，粘贴前执行 `:set paste`。**不要**把 markdown 的 ` ```cnf ` 行粘贴进文件。

参考配置内容如下：

```cnf

> **mysqld 启动报错 `Found option without preceding group`**
>
> 说明 `/etc/my.cnf` 里有配置项没有写在 `[client]`、`[mysqld]` 等分组下面，或粘贴了 ` ``` ` 等非法行。
>
> ```bash
> # 查看配置文件是否有异常行
> cat -A /etc/my.cnf
> # 重新写入正确配置后，重置失败状态再启动
> systemctl reset-failed mysqld
> systemctl start mysqld
> ```

##### 5.修改密码

yum 安装的 MySQL 会在日志中生成 root 临时密码，需先用临时密码登录，再改成自己的密码。

```bash
# 查看临时密码
grep 'temporary password' /var/log/mysqld.log
# 示例输出：root@localhost: Y-9x_&8TEkr+
```

**重要：** MySQL 5.7 默认开启密码强度校验（`validate_password`），纯数字如 `12345678` **不符合策略**，改密会失败：

```
Your password does not satisfy the current policy requirements
```

改密失败后 root 仍是**临时密码**，用新密码登录会报 `Access denied`。

```bash
# 方式一：mysqladmin 修改（密码需含大小写字母、数字、特殊字符，长度≥8）
# 临时密码含 & 等特殊字符，必须用单引号包裹
mysqladmin -uroot -p'Y-9x_&8TEkr+' password 'Admin@123'

# 方式二：先登录再修改（推荐，更直观）
mysql -uroot -p'Y-9x_&8TEkr+'
```

登录后在 MySQL 里执行：

```sql
-- 修改 root 密码（示例：Admin@123）
ALTER USER 'root'@'localhost' IDENTIFIED BY 'Admin@123';
exit;
```

若必须使用简单密码，可先降低策略（仅学习环境建议）：

```sql
mysql -uroot -p'临时密码'
SET GLOBAL validate_password_policy=LOW;
SET GLOBAL validate_password_length=8;
ALTER USER 'root'@'localhost' IDENTIFIED BY '12345678';
exit;
```

验证新密码：

```bash
mysql -uroot -p'Admin@123' -e "SELECT 1;"
```

### 四、部署tomcat和jdk

##### 1.创建包存放目录

```bash
mkdir -p /opt/soft/tomcat && cd /opt/soft/tomcat

# 方式一：虚拟机直接下载（推荐，使用 Apache 官方归档地址）
wget https://archive.apache.org/dist/tomcat/tomcat-8/v8.5.70/bin/apache-tomcat-8.5.70.tar.gz

# 若官方源较慢，可尝试阿里云镜像
# wget https://mirrors.aliyun.com/apache/tomcat/tomcat-8/v8.5.70/bin/apache-tomcat-8.5.70.tar.gz

# 注意：清华源 mirrors.tuna.tsinghua.edu.cn 对 Tomcat 可能返回 403，勿用

# 方式二：Windows 浏览器下载后，用 Xftp/WinSCP 上传到 /opt/soft/tomcat/

# 确认下载成功（约 10~11MB）
ls -lh apache-tomcat-8.5.70.tar.gz
```

##### 2.解压安装包

```bash
tar -zxvf apache-tomcat-8.5.70.tar.gz
```

##### 3.创建解压安装后的目录

```bash
mkdir /usr/local/tomcat8
```

##### 4.移动解压好的文件

```bash
mv /opt/soft/tomcat/apache-tomcat-8.5.70/* /usr/local/tomcat8/
```

##### 5.安装jdk

```bash
#创建目录并上传包
mkdir /opt/soft/jdk && cd /opt/soft/jdk
#创建Java文件夹
mkdir /usr/local/java
#上传jdk后解压
tar -zxvf jdk1.8.0_121.zip
#移动解压后的jdk
mv jdk1.8.0_121 /usr/local/java/
#授权，这里不授权java命令执行时会提示没权限
chmod +x /usr/local/java/jdk1.8.0_121/bin/*
chmod +x /usr/local/java/jdk1.8.0_121/jre/bin/*
```

##### 6.配置环境变量

```bash
#vim /etc/profile
export JAVA_HOME=/usr/local/java/jdk1.8.0_121
export JRE_HOME=${JAVA_HOME}/jre
export CLASSPATH=$JAVA_HOME/lib:${JRE_HOME}/lib:$CLASSPATH
export CATALINA_HOME=/usr/local/tomcat8
export JAVA_PATH=${JAVA_HOME}/bin:${JRE_HOME}/bin:${CATALINA_HOME}/bin
PATH=$PATH:$HOME/bin:$JAVA_HOME/bin:$CATALINA_HOME/bin
#刷新环境变量
source /etc/profile
```

##### 7.在Tomcat中配置jdk

```bash
vim /usr/local/tomcat8/bin/catalina.sh
#添加
export JAVA_HOME=/usr/local/java/jdk1.8.0_121/
export JRE_HOME=/usr/local/java/jdk1.8.0_121/jre
```

##### 8.启动Tomcat

```bash
[root@localhost /opt/soft/jdk]# cd /usr/local/tomcat8/bin/
[root@localhost /usr/local/tomcat8/bin]# ./startup.sh 
Using CATALINA_BASE:   /usr/local/tomcat8
Using CATALINA_HOME:   /usr/local/tomcat8
Using CATALINA_TMPDIR: /usr/local/tomcat8/temp
Using JRE_HOME:        /usr/local/java/jdk1.8.0_121/jre
Using CLASSPATH:       /usr/local/tomcat8/bin/bootstrap.jar:/usr/local/tomcat8/bin/tomcat-juli.jar
Using CATALINA_OPTS:   
Tomcat started.
```
